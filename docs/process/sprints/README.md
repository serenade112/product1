# スプリント運用ガイド

- 最終更新: YYYY-MM-DD
- スプリント長: 例) 2 週間（開始: 月 / 終了: 金）
- 管理: スプリントゴールと PBI/SBI は GitHub Projects を正本にし、docs は要約と振り返りのみ。

## スプリントの流れ
- Sprint Planning (45–60min)
	- ゴールを 1 文で定義
	- 対象 PBI を選定 (#123 形式で列挙)・見積・優先度
	- リスク/前提を確認
- Daily Scrum (15min)
	- 昨日/今日/障害を Projects 上で共有、ブロッカーはコメントに記録
- Backlog Refinement (週 1)
	- 次スプリント候補の PBI を DoR に沿って整備
- Sprint Review (30min)
	- デモ・成果・フィードバックを要約
- Retrospective (30min)
	- Keep / Problem / Try を 1 行ずつ決め、次スプリントの改善アクションに反映

## アーティファクト
- スプリントゴール: Planning で決定し、ノートに記録
- バーンダウン: Projects のビューを参照（docs にはリンクのみ）
- スプリントノート: ゴール、対象 PBI、デモ結果、振り返り、次の Try を記載

## スプリントノートの運用
- ファイル名: `sprint-YYYYWW.md`
- テンプレ: [sprint-YYYYWW-template.md](sprint-YYYYWW-template.md)
- 記載項目: ゴール 1 文、期間、対象 PBI、リスク、レビュー結果、K/P/T、次の Try
